The Weekly Toolbox (WN31)

This is The Weekly Toolbox, week 31 of 2021. We have three Go programs and a security framework document.

Dasel

A command line program written in GO for editing, transforming, and converting various data interchange formats. According to the feature set it can handle JSON, TOML, YAML, XML, and CSV. It’s an alternative to jq or yq. I think those are more popular in this category of tools. The project repository has a GIF loop to demo its functionality.

hex

Currently designing a REST API for another project so a new tool that may help with testing HTTP in Go is of interest. According to the project page, it extends httptest.Server for mocking APIs. This is the example from the repository README:

func TestUserClient(t*testing.T) {
	// A mock of some remote user service
	server := hex.NewServer(t, nil)	 // nil, or optional http.Handler

	server.ExpectReq("GET", "/users").
		WithHeader("Authorization", "Bearer xxyyzz")
		WithQuery("search", "foo").
		RespondWith(200, `{"id": 123, "name": "test_user"}`)

	// Actual client implementation would go here
	http.Get(server.URL + "/foo")

	// Output:
	// example_test.go:12: One or more HTTP expectations failed
	// example_test.go:12: Expectations
	// example_test.go:12: 	GET /users with header matching Authorization="Bearer xyz"query string matching search="foo" - failed, no matching requests
	// example_test.go:12: Unmatched Requests
	// example_test.go:12: 	GET /foo
}

JSON support is being improved but still looks good as it is right now. A good addition to my current project’s import list. Looking forward to trying it out.

Gatus

Grafana+Prometheus is the most popular in the category of dashboard toolkits right now but there’s room for smaller and targeted tools. Gatus is a health dashboard that can monitor services using HTTP, ICMP, TCP, and DNS endpoints. I have done a similar effort in Grafana+Prometheus but setting it up in Gatus is definitely easier.

Notifications go to Slack, PagerDuty, Discord, Telegram, or Twilio. The configuration is done through YAML and there’s an API. The frontend dashboard was done in Vue.js and is just enough for what it was designed for. Deploy seems simple according to the Dockerfile.

OWASP Application Security Verification Standard

The standard is freely available as PDF at their Github repository. I have not read it completely yet but it’s a framework to assist in designing and developing web applications or services.

The standard provides a basis for designing, building, and testing technical application security controls, including architectural concerns, secure development lifecycle, threat modelling, agile security including continuous integration / deployment, serverless, and configuration concerns.

At 60+ pages, seems a good addition to complement other frameworks or standards.
